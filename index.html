Creating an App with Sinatra - March 14, 2016

Recently I had the opportunity to create a Sinatra app for the coding bootcamp I'm enrolled in. Since running is one of my interests outside of coding, I decided to create a basic Sinatra app to keep track of a user's workout activity. To do this I used the classic MVC (models, views, controllers) design pattern which is oft used in these kinds of applications. I used ActiveRecord to make two databases, one for user data and one for workout data. The users table stores an individual's username, email, and an encrypted password for their account. The workout table has keys for the date/time of the workout, distance, intensity, and a foreign key corresponding to the user who created the workout instance.

I created three controllers, a main controller in which I enabled sessions and defined two helper methods (#logged_in? and #current_user) and two controllers, one for users and one for workouts, which inherited from my main application controller. I like splitting the controllers in this way because it contributes to code readability and follows the single responsibility principle. I use the two conrollers to build out that basic Sinatra CRUD actions (create, read, update, and destroy). Users can sign up for an account, add and persist workouts, view their workouts over time, edit or delete a specific workout.

I added some basic CSS styling with Bootstrap. I'm not 100% satisfied with the way the app looks yet, but overall I am satisfied that my app has has the basic functionality I wanted it to have. For future updates I plan to improve the front-end styling, and incorporate responsive design elements. Feel free to check out my code on Github.
